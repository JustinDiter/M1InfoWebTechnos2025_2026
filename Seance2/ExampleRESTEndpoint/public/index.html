Preset Manager for an audio sampler

This is a simple index.html HTML page that describes the REST API available on this server.

You should modify this file to create a simple client for the presets API.

Current API :

  • GET /api/health
  • GET /api/presets
  • GET /api/presets/:name. Example: /api/presets/electronic should return the preset JSON file with the name "electronic.json" located in the public/presets folder.

Question 4: what is the purpose of the tests folder?

The tests folder is used to store unit tests and integration tests for the application. These tests help ensure that the code behaves as expected and allows for easier maintenance and refactoring in the future.

On the back-end part:

You should implement the following features in the REST API located in the src/app.mjs file:

  1. Implement the GET /api/presets endpoint in index.mjs
  2. Implement the GET /api/presets/:name endpoint in index.mjs

For the first one: GET /api/presets endpoint, you should return the list of JSON files located in the public/presets folder as an array of objects.

For the second one: GET /api/presets/:name endpoint, you should return the content of the preset JSON file as an object.

On the front-end part:

You should create a simple client for this API in this index.html file.

Use the fetch API to call these endpoints and display the results in this page.
